## ВҝQuГ© es esto?

**TJ Simulator** es un ecosistema completo para el desarrollo de un robot **Micromouse** competitivo, creado en la **Universidad AnГЎhuac Mayab** para la competencia **ROBGAM 2026**. El proyecto une dos mundos en un solo repositorio:

Un **simulador fГӯsico en Python/Pygame** que replica fielmente el comportamiento del robot вҖ” motores GA12-N20 con encoders duales de distinto CPR, IMU MPU6500 con integraciГіn de gyro, sensores ToF VL53L0X con raycast DDA, y PID de triple lazo вҖ” todo animado a 60 fps con fГӯsica continua desacoplada del timer lГіgico.

Un **firmware PlatformIO para ESP32-S3** que implementa los mismos algoritmos del simulador: Flood Fill exploratorio (Run 1), A* Гіptimo (Run 2) y A* a velocidad mГЎxima (Run 3), con mapa persistente en EEPROM y telemetrГӯa por Serial que puede reproducirse en el simulador fotograma a fotograma.

El laberinto estГЎndar de competencia es **16Г—16 celdas de 18 cm**, con robot de mГЎximo **10Г—10 cm**. Los motores operan a ~67 RPM efectivos (100 RPM @ 6V, alimentados a 4V), cubriendo una celda en ~2.2 s durante exploraciГіn.

ESTE SOFTWARE SOLO SE UTILIZГ“ PARA OPTIMIZAR LOS ALGORITMOS PREVIOS A TENER UN LABERINTO FГҚSICO. AL MOMENTO DE LA COMPETICIГ“N SE MANTIENE DESCONECTADO EN TODO MOMENTO.

### Estrategia de competencia вҖ” 3 runs

```
Run 1 вҶ’ Flood Fill    VEL_EXPLORE=170   Explorar laberinto + guardar mapa EEPROM
          вҶ“  (al llegar a meta: calcula A*, regresa al inicio)
Run 2 вҶ’ A*            VEL_FAST=215      Camino Гіptimo descubierto en Run 1
          вҶ“  (regresa al inicio)
Run 3 вҶ’ A*            VEL_SPRINT=240    Mismo camino a mГЎxima velocidad
```

> Si el robot se apaga entre runs, la EEPROM conserva el mapa. Al encender de nuevo salta directamente a Run 2.

## TelemetrГӯa

### Protocolo Serial ESP32 вҶ’ Simulador

El firmware envГӯa **una lГӯnea por waypoint** (cada FWD, TURN o GOAL):

```
TEL,<ts_ms>,<event>,<x>,<y>,<heading>,<ang_z>,<enc1>,<enc2>,<izq>,<cen>,<der>
```

| Campo | DescripciГіn |
|---|---|
| `ts_ms` | Timestamp desde arranque del ESP32 (ms) |
| `event` | `FWD` / `TURN_L` / `TURN_R` / `TURN_180` / `GOAL` / `BOOT` |
| `x`, `y` | PosiciГіn lГіgica en el laberinto |
| `heading` | 0=N В· 1=E В· 2=S В· 3=W |
| `ang_z` | ГҒngulo integrado del gyro (grados) |
| `enc1`, `enc2` | Counts acumulados de M1 y M2 |
| `izq`, `cen`, `der` | Lecturas VL53L0X en mm |

### Replay dual (SIM vs ESP32)

```
1. Tab REPLAY вҶ’ [ ] GRABAR вҶ’ ejecutar simulaciГіn вҶ’ detener grabaciГіn
2. Correr el robot fГӯsico вҶ’ recoger CSV de telemetrГӯa
3. Cargar Sim (cyan) + Cargar ESP32 (naranja)
4. Seleccionar canal: SIM / ESP32 / AMBOS
5. >> REPRODUCIR вҖ” ambos robots animados simultГЎneamente
```

El robot **naranja (ESP32)** se superpone sobre el **cyan (Simulador)** en Modo Fantasma, interpolando entre waypoints con ease-out para mostrar exactamente dГіnde divergieron las decisiones y tiempos.
